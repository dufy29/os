实验框架代码与提交方法

课程所有 Lab 代码均托管在同一 git repository 中，并且进度全程追踪。请不要去除 Makefile 中 Git tracing 的部分。

实验环境

实验在须使用 64 位 Linux 平台，强烈推荐在真机上安装。Debian/Ubuntu 是比较好的选择。

截止日期

  • 由于两个班级进度相差巨大，不设置具体 deadline，请大家自觉安排实验。Hard deadline 参见课程主页。
  • 提交需校园网、配置环境变量。请预留好足够时间。提交时间以最新提交在服务器上保存的时间戳为准。
  • 服务器时间已与北京时间同步，可能存在少量误差，但仍以服务器时间为准。
  • 如提交截止前服务器故障，提交时间将顺延。

获取实验框架，在终端中执行

git clone https://github.com/NJU-ProjectN/ics-workbench.git

将会克隆 ics-workbench 到当前目录，之后的编程任务均在此完成。请妥善保管这个目录。如遇问题请联系老师或助教。

提交

配置好STUID和STUNAME环境变量，在实验对应目录中执行make submit，将会自动提交到对应的实验。例如在 multimod/ 目录中 make submit 将会提交到 Lab1。
