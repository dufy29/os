课程简介

大家对“研究生”的生活一定感到向往又有些迷茫。与本科时上课学习不同，研究生的培养目标是解决 (甚至是提出) 世界上未曾解决过的问题。《软件工程研究导引》的教学目标是使学生熟悉软件工程研究的 “three easy pieces”：读论文 (reading)、写论文 (writing)、写代码 (hacking)，做好从本科生到研究生的过渡。

我们预期这门课的参与者有良好的编程基础 (C/C++ 和 Java)，熟练使用 Linux 系统工具，并对软件工程实践有一些自己的理解。

本课程页面永久链接：http://ics.nju.edu.cn/~jyywiki/wiki/GSER2019 (校园网)

  • 任课教师：许畅、蒋炎岩、马骏

软件工程研究导引：主题

  • R (Reading)：在软件工程相关的主题下阅读并报告论文，每位参与的同学都需要阅读所有论文。课程中关注的五个主题 (主题并不严格正交)：
    • Empirical software engineering (EMSE); 实证软件工程研究
    • Software engineering process (SEP); 软件工程过程
    • Formal methods (FM); 形式化方法
    • Software testing and analysis (STA); 软件测试与分析
    • Software maintenance and evolution (SME); 软件维护与演化
  • W (Writing)：科技写作方法与实践。
  • H (Hacking)：软件工程研究中的常用的系统编程工具分析软件，配有一定量的编程作业 (programming assignments)。

阅读列表

我们努力地筛选出那些我们认为重要、经典、有 long term impact 的好论文，这代表了我们对 “research significance” 的理解和追求。如果你对论文列表有补充或有更好的建议，请联系我们！

在开始之前，我们建议你阅读 How to read a paper，其中的 three-pass approach 也是我们所推荐的。你能在互联网上找到更多教授给阅读论文和做研究提出的建议。阅读论文时请留意：

  1. 论文解决什么样的问题？从软件工程实践者的角度，这个问题在软件工程实践中有怎样的意义？这是个新问题，还是老问题？这个问题为什么 significant？
  2. 如果是你解决这个问题，会是怎样的？论文提出了怎样的解决方法？
  3. 这个方法有什么 novelty？做到了怎样的效果？
  4. 几乎任何东西都有它们的 limitations，你能找到吗？

最后，虽然论文基本都做到 self-contained，但你有时需要阅读一些额外的参考文献补充必要的背景知识。

Reading Session 的要求

对于每个主题 (EMSE, SEP, FM, STA, SME)，每个同学需要准备好指定一篇论文的报告 (提交 slides，提交方法待定)。我们将会在课上随机选取一位同学进行报告。此外，所有同学需要阅读所有论文。详情请阅读这里。

每个同学在每个论文主题下输入学号查看需要准备报告的论文。
