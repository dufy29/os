加入软件所大家庭

蒋炎岩来自南京大学计算机软件研究所的 SPAR (System and Program Analysis Research) 小组。软件所有多个研究方向，加入课题组的同学可以根据自己的研究兴趣联系指导老师 (发邮件即可)。

  • 本科生 (Research Intern) 🙇‍♂️
  • 硕士/博士研究生 (申请攻读学位) 👨‍🎓

为什么要做软件方面的研究？

如今，软件已经像水、电、Wifi 一样成为人类生活不可缺少的基础设施。大家都没少吃过编程的苦，也一定体会过编程的乐趣——你是否发自内心地想拯救水深火热中的码农们，帮助他们又快又好地生产软件？

JYY 认为软件方面的研究同时兼顾了“追求真理” (for fun)、“创造价值” (for profits)。一方面解决软件问题会涉及编程语言、计算机系统中的本质问题，另一方面软件方面的研究能做出真正实用的成果，回馈开源社区乃至整个社会。

无论你在哪里做研究，起步时都会相当迷茫。因此我们准备了一些材料，帮助大家在找不到方向时重温一下“我们应该做什么”。

入门和基本功

  • 计算机系统相关的阅读材料。只要你觉得非常热爱编程，那就足够了！
  • 组内代码和论文都使用 Git 管理，因此有必要阅读 Git Best Practices。
  • 做实验：Benchmarking crimes。

写作

“The Elements of Style” 是一本很棒的小册子。如果你不是写作的老手，在读它的时候一定是不断在中枪的。网上也有很多有用的资料：常见写作问题 by Tao Xie 和知乎大神的帖子。

此外，你需要会用 LaTeX，并且绝对有必要知道 LaTeX 里一些容易犯错误的地方。此外，表格、流程图、Plotting 等都有相应的技巧——你经常能读到让你赏心悦目的论文，也一定会想写得和它们一样好。最后，用什么能在两周内搞定 150 页的毕业论文？南京大学山寨 (LyX) 毕业论文模板。

在实际写作时，你会用到一些写作工具，例如给你的工具起一个好的缩写。
